1. ... v Pythonu: to není lenost, ale produkční kód

V případě, že uvidíte následující definice funkcí (zapsané v různých jazycích), pravděpodobně budete předpokládat, že tyto funkce nejsou dokončeny a že zdrojové kódy s těmito funkcemi nebudou přeložitelné nebo spustitelné:

void foo() {
   ...
}
 
fn foo() {
   ...
}
 
func foo() {
   ...
}
 
def foo():
   ...

V prvních třech případech (což je konkrétně C, Rust a Go) budete mít naprostou pravdu – takto zapsané funkce nejenom že neobsahují tělo, ale navíc jsou v nich zapsány „trojtečky“ (ellipsis), které nejsou korektním literálem. Ovšem v případě posledním je tomu jinak. Poslední funkce je totiž zapsána v Pythonu a tento jazyk má ... definován jako literál. Pochopitelně jsou stanovena i syntaktická a sémantická pravidla použití tohoto literálu. A právě s těmito pravidly a způsoby použití se setkáme v dnešním článku. Znaky ... se totiž v Pythonu mohou použít na více místech, mnohdy tam, kde bychom je vlastně ani nečekali. A navíc můžeme pomocí ... vyřešit některé zapeklitější sémantické problémy, například schopnost zjistit, zda byl do funkce předán parametr.

2. Objektový systém jazyka Python a literál ...

Při spuštění interpretru jazyka Python je (kromě dalších hodnot) inicializována i hodnota uložená do proměnné Ellipsis. Tato hodnota je jedináčkem (singleton) a je typu ellipsis:

help(Ellipsis)

Měla by se zobrazit nápověda ke třídě ellipsis:

class ellipsis(object)
 |  The type of the Ellipsis singleton.
 |
 |  Methods defined here:
 |
 |  __reduce__(self, /)
 |      Helper for pickle.
 |
 |  __repr__(self, /)
 |      Return repr(self).
 |
 |  ----------------------------------------------------------------------
 |  Static methods defined here:
 |
 |  __new__(*args, **kwargs)
 |      Create and return a new object.  See help(type) for accurate signature.

O existenci jedináčka Ellipsis se ostatně můžeme velmi snadno přesvědčit:

print(Ellipsis)

Mělo by se vypsat:

Ellipsis

Důležité je, že literál ... je nastaven na stejnou hodnotu:

print(Ellipsis)
print(...)

V tomto případě se vypíšou dva naprosto stejné řádky:

Ellipsis
Ellipsis

To ovšem nic neříká o tom, že jsou obě hodnoty stejného typu. Ověření musí proběhnout odlišně:

print(type(Ellipsis))
print(type(...))

<class 'ellipsis'>
<class 'ellipsis'>

Navíc se snadno přesvědčíme o tom, že se jedná nejenom o stejné hodnoty, ale i o shodný objekt ležící ve stejném místě operační paměti:

print(id(Ellipsis))
print(id(...))
print(id(Ellipsis) == id(...))

9918048
9918048
True

Poznámka: vypsaná čísla se velmi pravděpodobně budou ve vašem případě odlišovat, ale oba první řádky budou totožné a poslední řádek bude obsahovat hodnotu True.

3. Je tedy ... pouze jmenným aliasem pro Ellipsis?

Z příkladů uvedených v předchozích kapitolách by se mohlo zdát, že trojice teček ... je pouze jmenným aliasem pro proměnnou Ellipsis. Ovšem to není přesné, protože s Ellipsis je možné skutečně pracovat jako s běžnou proměnnou, což mj. znamená, že je do ní možné přiřadit novou hodnotu. Ostatně si to můžeme velmi snadno ukázat:

print(Ellipsis)
print(Ellipsis == ...)
print()
 
Ellipsis=42
print(Ellipsis)
print(Ellipsis == ...)

Po spuštění tohoto skriptu se nejdříve vypíše původní hodnota přiřazená do proměnné Ellipsis i to, zda je tato hodnota totožná s hodnotou reprezentovanou literálem ...:

Ellipsis
True

Ovšem další řádky vypsané skriptem prozradí, že nám prakticky nic nebrání v tom, abychom hodnotu uloženou do proměnné Ellipsis přepsali jinou hodnotou:

42
False

Poznámka: proměnné ve skutečnosti obsahují reference na hodnoty.

Ovšem literál ... se chová jinak než běžná proměnná. To především znamená, že ho není možné použít na levé straně přiřazovacího příkazu:

print(Ellipsis)
print(Ellipsis == ...)
print()
 
...=42
print(Ellipsis)
print(Ellipsis == ...)

Interpret programovacího jazyka Python v tomto případě vypíše chybu (dokonce chybu na úrovni syntaxe) již při pokusu o načtení zdrojového kódu:

  File "/home/ptisnovs/src/most-popular-python-libs/ellipsis/ellipsis_assignment_2.py", line 5
    ...=42
    ^^^
SyntaxError: cannot assign to ellipsis here. Maybe you meant '==' instead of '='?

5. Definice funkce s dočasným tělem

Připomeňme si, že v Pythonu se nepoužívají klasické příkazové bloky známé z jiných programovacích jazyků. Namísto toho blok začíná dvojtečkou a kód bloku musí být odsazen (a typicky ukončen prázdným řádkem). Jak se ovšem v takovém případě zapíše „prázdný blok“. Většinou se setkáme s použitím klíčového slova pass:

def foo():
   pass

Ovšem naprosto stejně můžeme do těla funkce zapsat jakoukoli hodnotu (libovolného typu):

def foo():
   42
 
 
print(foo())

Po spuštění je ověřeno, že 42 má význam pouze ze syntaktického hlediska. Tato hodnota je vyhodnocena, ale není vrácena.

None

Klíčové slovo pass se používá tehdy, pokud funkci již nechceme dále měnit – je to její finální podoba (některé funkce nebo metody skutečně pouze musí existovat, ale mohou mít prázdné tělo). Pokud ovšem budete chtít naznačit, že se funkce ještě v budoucnu rozšíří, je lepší použít tento způsob zápisu:

def foo():
   ...

Poznámka: jedná se o zcela korektní funkci, kterou je možné zavolat. Literál ... je zde použit stejně tak, jako jsme dříve použili hodnotu 42. Žádný jiný speciální význam nemá.
