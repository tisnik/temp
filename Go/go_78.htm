1.

V seriálu o programovacím jazyce Go jsme se již zmínili o možnosti zkombinování tohoto staticky typovaného a překládaného programovacího jazyka s nějakým interpretrem, typicky s interpretrem vyššího programovacího jazyka. Díky této kombinaci je například možné „skriptovat“ aplikaci naprogramovanou v Go, rozšiřovat tuto aplikaci o další logiku atd. Většinou je tato možnost rozšiřování možností aplikace s využitím skriptů výhodná pro obě strany – firma produkující daný software vlastně zadarmo získá další aplikační oblast a zákazník či uživatel si takto může aplikaci upravit ke svým potřebám.

Prozatím jsme se věnovali způsobům vestavění interpretru programovacího jazyka Lua do aplikace naprogramované v Go. Kombinace Lua+Go resp. obecněji Lua+staticky typovaný překládaný jazyk je velmi běžná a vlastně i logická, protože právě jazyk Lua je navržen s ohledem na jeho relativně snadnou „vložitelnost“ (embed) do větších aplikací. Důvodem, proč jsou některé hry, například Escape from Monkey Island, Grim Fandango, Fish Fillets, Neverwinter Nights či MDK2 z menší či větší části naprogramované právě v Lue, spočívá v tom, že kombinace nízkoúrovňového a skriptovacího jazyka umožňuje soustředit se při vývoji na podstatné věci – herní engine vytvořit co nejefektivnější s využitím všech možností nízkoúrovňového jazyka a naopak herní scénář a logiku hry naskriptovat s co největším urychlením cyklu oprava–překlad–spuštění.

Poznámka: ve skutečnosti existuje hned několik projektů, které programátorům používajícím jazyk Go zpřístupňují interpret skriptovacího jazyka Lua. Jedná se o projekty Gopher-Lua, go-lua a taktéž golua (interface k původnímu céčkovému interpretru). Rozšířením knihovny Gopher-Lua je Binder.

package main
 
import (
        "fmt"
        "github.com/PaesslerAG/gval"
)
 
func main() {
        vars := map[string]interface{}{"name": "World"}
        value, err := gval.Evaluate(`"Hello " + name + "!"`, vars)
        if err != nil {
                fmt.Println(err)
        }
 
        fmt.Print(value)
}
